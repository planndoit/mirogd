# 지도/캐릭터 디자인 개선 옵션

## 1. PixiJS (권장)

- **특징**: WebGL 기반 2D 렌더러, 선·도형이 매우 매끄럽고 성능 좋음. Phaser가 내부적으로 사용.
- **번들**: 약 450KB (minified). Canvas만 쓰는 현재보다 크지만 체감 품질 향상이 큼.
- **적합**: 지도는 `Graphics`로 벽선(round join), 캐릭터는 원+그라데이션 또는 간단 스프라이트. **위치 보간(lerp)**으로 이동을 부드럽게 표현 가능.
- **참고**: https://pixijs.com/

## 2. Phaser

- **특징**: 게임 프레임워크(씬, 물리, 타일맵, 스프라이트 등). 내부 렌더링은 PixiJS.
- **번들**: 약 1MB+. 미로를 타일맵으로 쓰고 캐릭터를 스프라이트로 두기 좋음.
- **적합**: 나중에 이펙트·애니메이션·사운드까지 한 번에 넣고 싶을 때.

## 3. 라이브러리 없이 Canvas만 개선

- **개선 포인트**:
  - `ctx.lineJoin = 'round'`, `lineCap = 'round'` 로 벽선 모서리 부드럽게
  - 캐릭터: 그라데이션 원 + `shadowBlur` 로 입체감
  - 서버 위치를 받을 때 **클라이언트에서 lerp** 해서 부드럽게 이동
- **장점**: 의존성 없음, 번들 크기 유지.

---

**현재 적용**: PixiJS로 게임 뷰를 전환해 지도·캐릭터를 매끄럽게 표현합니다.
