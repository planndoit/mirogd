# 미로 속 경찰과 도둑 (MVP)

## 1. 프로젝트 개요

**미로 속 경찰과 도둑**은 친구에게 **방 링크를 공유**해 함께 입장한 뒤,  
**랜덤 생성된 2D 미로**에서 경찰과 도둑으로 나뉘어 추격전을 벌이는  
웹 기반 실시간 멀티플레이 게임.

이 프로젝트는 복잡한 기능보다 **미로에서 추격하는 재미 자체를 검증하는 MVP**에 집중한다.

핵심 특징:

- 회원가입 없음
- 닉네임 기반 참여
- 링크 공유로 방 입장
- 랜덤 미로 생성
- 실시간 멀티플레이
- 경찰 vs 도둑 추격전
- 탈락 후 관전

### 실행 방법

의존성 설치 (최초 1회):

```bash
npm run install:all
```

터미널 2개에서 각각 실행:

```bash
# 터미널 1: 게임 서버 (기본 포트 3001)
npm run dev:server

# 터미널 2: 웹 클라이언트 (기본 포트 3000)
npm run dev:client
```

브라우저에서 http://localhost:3000 접속 후 방 만들기 또는 방 링크로 입장.

## 3. 핵심 게임 컨셉

이 게임은 **숨바꼭질 게임이 아니라 전략 추격 게임**이다.

특징:

- 모든 플레이어는 **맵 전체를 볼 수 있음**
- 모든 플레이어의 **위치가 공개됨**
- 이동 속도 동일
- 미로 구조를 이용한 **경로 예측과 차단 전략**

---

## 4. 사용자 흐름

### 1. 첫 화면

사용자는 다음 중 하나를 선택한다.

- 방 만들기
- 방 링크로 입장

### 2. 방 생성

사용자는 닉네임을 입력하고 방을 생성한다.

설정 항목:

- 닉네임
- 최대 게임 인원
- 게임 시간

방 생성 후 고유 링크 생성:

`/room/{roomId}`

예:

`example.com/room/abc123`

이 링크를 친구에게 공유한다.

### 3. 방 입장

사용자는

1. 닉네임 입력
2. 방 입장

다음 규칙 적용

- 플레이어 수가 최대 인원 미만 → **플레이어로 참여**
- 최대 인원 초과 → **관전자로 입장**
- 게임 진행 중 입장 → **관전자로 입장**

## 5. 방 대기실

대기실에서는 다음 정보를 표시한다.

### 플레이어 목록
현재 게임 참가자

### 관전자 목록
관전 중인 사용자

### 설정 정보
- 최대 인원
- 게임 시간

### 방장 기능

방장은 다음 설정을 변경할 수 있다.

- 최대 인원
- 게임 시간

그리고

**게임 시작 버튼**을 누를 수 있다.

---

## 6. 게임 시작 흐름

### 1. 랜덤 미로 생성

게임 시작 시 플레이어 수에 맞게 **랜덤 미로 생성**

예:

| 플레이어 수 | 맵 크기 |
|---|---|
| 3명 | 소형 |
| 4~5명 | 중형 |
| 6명 이상 | 대형 |

미로 생성 알고리즘:

- DFS 기반 미로 생성

### 2. 랜덤 스폰

플레이어들은 미로 통로 위 **랜덤 위치에 스폰**

이 시점에는 역할 없음.

### 3. 준비 시간 (10초)

플레이어는 자유롭게 이동 가능

목적:

- 맵 파악
- 전략 준비

### 4. 역할 랜덤 배정

10초 후 역할이 랜덤으로 배정된다.

역할:

- 경찰
- 도둑

역할 비율은 플레이어 수에 따라 자동 조정.

## 7. 게임 규칙

### 시야

모든 플레이어는

- 맵 전체 확인 가능
- 모든 플레이어 위치 확인 가능

### 이동

자유 이동 방식.

**PC**

방향키 ↑ ↓ ← →

**모바일**

화면 하단 방향 버튼 ↑ ↓ ← →

캐릭터는 **벽을 통과할 수 없음**.

### 이동 속도

경찰과 도둑 **이동 속도 동일**

### 잡기 판정

경찰 캐릭터와 도둑 캐릭터 사이 거리가 일정 거리 이하이면

→ 도둑 잡힘

### 탈락

도둑이 잡히면

- 즉시 탈락
- 관전 모드 전환

### 닉네임 표시

각 캐릭터 위에 닉네임 표시.

---

## 8. 승리 조건

### 경찰 승리
모든 도둑 체포

### 도둑 승리
제한 시간 생존

---

## 9. 게임 종료 후 흐름

게임 종료

→ 결과 화면 표시

→ 자동으로 **방 대기실로 복귀**

대기실에서

- 방장은 설정 변경 가능
- 다시 게임 시작 가능

---

## 10. 관전 시스템

관전자는 다음과 같다.

- 최대 인원 초과 입장
- 게임 중간 입장
- 탈락한 플레이어

관전자는

- 게임 화면 실시간 시청 가능
- 다음 판 참여 가능

## 13. MVP 목표

이 프로젝트의 목표는

**친구들과 플레이했을 때 "한 판 더 하자"라는 반응이 나오는지 검증하는 것**이다.

핵심 검증 요소:

- 미로 크기
- 이동 속도
- 플레이어 수
- 게임 시간
